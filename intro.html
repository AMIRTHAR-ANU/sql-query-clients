Introduction¶

ibmcloudsql¶

Working with structured data in IBM Cloud Object Storage using SQL language, involves multiple capabilities:

    1. IBM Cloud access which control access to all IBM catalogs/resources

    1. IBM COS instance where input and outpout data is stored.

    1. IBM Cloud SQL Query instance which offers the SQL-based data processing service

    1. Optionally: IBM Watson Studio: which provides notebook environment for running notebook, in that (Python) code and data asset are also stored, implicitly, in the Project’s IBM COS instance.

    1. Visualization: can be done with Python code via IBM Watson Studio’s notebook

This ibmcloudsql library provides these multiuple parts to Python applications. It comes with multiple submodules, each one links to another via subclass mechanism.

The submodules

  • utilities provides (A) - the IBM Cloud service connectivity-related functionality

  • cos provides (A, B) - the COS-related functionality

  • SQLQuery provides (A, B, C)

  • (D, E) is done via notebook-based examples of user-specific code which also utilizes above submodules. For getting started use this starter notebook.

To install your package in from source in this repository in editable mode:

cd sql-query-client/Python
pip install -e .

To run the test on the package, run either

python -m pytest

pytest

To install the published release of ibmcloudsql from PyPi:

pip install ibmcloudsql
